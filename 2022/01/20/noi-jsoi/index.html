NOI系列竞赛活动-江苏认证环境

$2021$ 年 $10$ 月 $19$ 日，中国计算机协会（CCF）在其全国青少年信息学奥林匹克竞赛（NOI）官网上发表了《关于CSP-J/S 2021第二轮各省认证环境的情况公布》一文。

根据文中信息显示，大多数省份都选择了 NOI Liunx 2.0 或 Windows 系统作为选手们的认证环境，也有不少省份选择了 Windows + NOI Liunx 2.0 虚拟机的形式。但江苏的认证环境情况有些别具一格：江苏并不使用 NOI Liunx 2.0 ，而是使用在 Windows 环境下搭建的 NOI Liunx 2.0 裁剪版。

什么是 NOI Liunx 2.0 裁剪版 呢？正巧，ChenLingHan 同学是一名 JSOIer，有幸参加了在南京举办的 $CSP2021$ 江苏赛区认证活动，并亲自在考场上体验了 NOI Liunx 2.0 裁剪版！

那么 NOI Liunx 2.0 裁剪版 到底是什么样的一个概念呢？优势又在哪里呢？对选手的认证有什么影响呢？下面请读者随着 ChenLingHan 同学的讲解来看看吧！

提供的编程工具

江苏CSP考场机子的桌面上有这么几个编程软件：

  • Dev-Cpp（GCC 4.9，默认开启 C++14，生成 32-bit 二进制文件）
  • Visual Studio Code（无配置，请自行配置）
  • GVim 8.2
  • Emacs 27.2

整场考试我只使用了Dev-Cpp（轻量级C++ IDE）。因为网络是断开的，所以 Visual Studio Code 没有C++包（悲）。

JSOI Liunx

江苏CSP考场机子的桌面除了几个编程软件之外，还有一个html的快捷方式叫做JSOI Liunx，这便是 NOI Liunx 2.0 裁剪版 的核心部分了。

点击快捷方式，IE浏览器会打开http://localhost:8080，界面如下：

NOI Liunx 2.0 下的界面

在比赛开始之前，选手需要提交代表身份的id.txt到该系统当中：

  • 第一行为机位号（例如 6B-015，S103-002）
  • 第二行为认证/考试号 （例如 JS-00123）

提交之后，原先请求提交id.txt的地方会变成你的准考证号和机位号，你后续提交的程序就会存储到你的名下（因为座位号和准考证号是无序的，所以不会产生有人提交别人的身份进行考试的情况）。

在比赛过程中，你所写的程序也是通过这个网页上传。比较人性化的一点是，这个网页系统会返回你的程序在 NOI Liunx 2.0 以及 CCF 公布的标准下的编译结果。

NOI Liunx 2.0 下的编译结果

江苏为程序运行设置了较小的内存、输出和时间限制，结果仅供参考。实际限制以题面和 CCF 公布的条例为准。

考试当中如果你编写的程序在该网页显示运行超时了，那么这个网页会不停的编译、运行你的程序（这也说明了该网页的运行原理）。此外，显示运行超时也不要慌，你仍然可以得到应有的暴力部分分。

此外，该网页还允许将输入文件（.in）上传。当存在同名.cpp 和.in 文件时，JSOI Linux 将会运行编译通过的程序，并展示输出结果（如图）：

NOI Liunx 2.0 下的输出结果

另外要注意的是，如果你的程序在该网页显示运行超时，那么这个输出文件的显示是一直在变化的（输出到限制运行时间结束的程序输出，然后清空，再输出到限制运行时间结束的程序输出。）我怀疑 JSOI Liunx 的工作原理就是不停的在编译运行这些程序。

最后，江苏增加了额外的编译选项，并在运行时检查数组越界，编译器的警告和越界检测结果供大家参考。错误检查对程序编译执行无影响，如图：

NOI Liunx 2.0 下的错误检查

好了，以上就是关于对江苏认证环境的全部内容了。简单的来说江苏认证环境就是为了方便选手编写程序并且让选手获得与正式评测相同的评测结果，防止出现了系统不同导致编译结果不同的问题，不得不说江苏认证环境的人性化！
